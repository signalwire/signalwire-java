# WebService Documentation

The `WebService` class (`com.signalwire.sdk.web.WebService`) provides static file serving capabilities for the SignalWire AI Agents SDK for Java. It can run as a standalone service or alongside your AI agents.

## Overview

WebService is designed to serve static files with configurable security features. It's perfect for:
- Serving agent documentation and API specs
- Hosting static assets (images, CSS, JavaScript)
- Serving generated reports and exports
- Providing configuration files and templates
- Hosting agent UI components

### Key Features
- **Multiple directory mounting** - Serve different directories at different URL paths
- **Security-first design** - Authentication, CORS, security headers, file filtering
- **HTTPS support** - SSL/TLS via `SecurityConfig` (env vars / config file)
- **MIME type handling** - Automatic content-type detection
- **Path traversal protection** - Prevents access outside designated directories
- **File filtering** - Allow/block specific file extensions

## Installation

```groovy
implementation 'com.signalwire:signalwire-sdk:4.0.0'
```

## Quick Start

```java
import com.signalwire.sdk.web.WebService;
import java.util.Map;

// Create a service to serve files
var service = new WebService();
service.addDirectory("/docs", "./documentation");
service.addDirectory("/assets", "./static/assets");

// Start the service (non-blocking); returns the bound port.
int port = service.start();
// Service available at http://127.0.0.1:8002
// Basic Auth: auto-generated by SecurityConfig
```

## Configuration

WebService can be configured through multiple methods (in order of priority):

### 1. Constructor Parameters

The full constructor is positional: `WebService(port, directories, basicAuth, configFile, enableDirectoryBrowsing, allowedExtensions, blockedExtensions, maxFileSize, enableCors)`.

```java
var service = new WebService(
    8002,                                       // Port to bind to
    Map.of(                                     // URL path to directory mappings
        "/docs", "./documentation",
        "/assets", "./static"),
    new String[] {"admin", "secret"},           // Custom basic auth [user, pass]
    null,                                        // configFile
    true,                                        // enableDirectoryBrowsing
    List.of(".html", ".css", ".js"),            // allowedExtensions (whitelist)
    List.of(".env", ".key"),                    // blockedExtensions (blacklist)
    100L * 1024 * 1024,                         // maxFileSize (100MB)
    true);                                       // enableCors
```

### 2. Environment Variables

```bash
# Basic authentication
export SWML_BASIC_AUTH_USER="admin"
export SWML_BASIC_AUTH_PASSWORD="secretpassword"

# SSL/HTTPS configuration
export SWML_SSL_ENABLED=true
export SWML_SSL_CERT_PATH="/path/to/cert.pem"
export SWML_SSL_KEY_PATH="/path/to/key.pem"
```

### 3. Configuration File

Create a `web.json` file (discovered automatically, or pass its path as the `configFile` constructor argument):

```json
{
    "service": {
        "port": 8002,
        "directories": {
            "/docs": "./documentation",
            "/api": "./api-specs",
            "/reports": "./generated/reports"
        },
        "enable_directory_browsing": true,
        "max_file_size": 52428800,
        "allowed_extensions": [".html", ".css", ".js", ".json", ".pdf"],
        "blocked_extensions": [".env", ".key", ".pem"]
    },
    "security": {
        "basic_auth": {
            "username": "admin",
            "password": "secure123"
        },
        "ssl_enabled": true,
        "ssl_cert": "/etc/ssl/certs/server.crt",
        "ssl_key": "/etc/ssl/private/server.key",
        "allowed_hosts": ["*"],
        "cors_origins": ["*"]
    }
}
```

## Security Features

### Basic Authentication

WebService implements HTTP Basic Authentication (timing-safe comparison via `MessageDigest.isEqual`). Credentials can be set via:

1. **Constructor**: `new String[] {"username", "password"}`
2. **Environment**: `SWML_BASIC_AUTH_USER` and `SWML_BASIC_AUTH_PASSWORD`
3. **Config file**: `security.basic_auth` section
4. **Auto-generated**: If not specified, `SecurityConfig` generates random credentials

### File Security

#### Default Blocked Extensions/Files
- `.env`, `.git`, `.gitignore`
- `.key`, `.pem`, `.crt`
- `.pyc`, `__pycache__`
- `.DS_Store`, `.swp`

#### Path Traversal Protection
WebService normalizes the resolved path and rejects anything outside the mounted base directory:

```
// These attempts return 403 Access denied:
// GET /docs/../../../etc/passwd
// GET /docs/./././../config.json
```

#### File Size Limits
Default maximum file size is 100MB. Configure it through the constructor's `maxFileSize` argument:

```java
var service = new WebService(
    8002, null, null, null, false, null, null,
    50L * 1024 * 1024,   // 50MB
    true);
```

### Security Headers

Automatically adds security headers (from `SecurityConfig.getSecurityHeaders()`) to all file responses:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Strict-Transport-Security` (when HTTPS is enabled)

## HTTPS/SSL Support

Unlike the JDK `start()` entry point (which binds plain HTTP), SSL is configured through `SecurityConfig` — driven by environment variables or the `security` section of the config file — not by arguments to `start()`.

### Method 1: Environment Variables

```bash
export SWML_SSL_ENABLED=true
export SWML_SSL_CERT_PATH="/path/to/cert.pem"
export SWML_SSL_KEY_PATH="/path/to/key.pem"
```

### Method 2: Configuration File

```json
{
    "security": {
        "ssl_enabled": true,
        "ssl_cert": "/etc/ssl/certs/server.crt",
        "ssl_key": "/etc/ssl/private/server.key"
    }
}
```

### Generating Self-Signed Certificates

For development/testing:

```bash
# Generate a self-signed certificate
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem \
    -days 365 -nodes -subj "/CN=localhost"

# Point the service at it
export SWML_SSL_CERT_PATH="cert.pem"
export SWML_SSL_KEY_PATH="key.pem"
```

## API Endpoints

### GET /{route}/{file_path}
Serve files from mounted directories.

**Parameters:**
- `route`: The mounted directory route (e.g., `/docs`)
- `file_path`: Path to file within the directory

**Response:**
- File content with the appropriate MIME type and security/cache headers
- `404` if the file is not found
- `403` if the file type is blocked, exceeds `maxFileSize`, or resolves outside the base directory
- `401` if basic auth is required and credentials are missing/invalid

A request that resolves to a directory is served that directory's `index.html`.

## Usage Examples

### Basic File Serving

```java
import com.signalwire.sdk.web.WebService;

// Serve documentation
var service = new WebService();
service.addDirectory("/docs", "./documentation");
service.addDirectory("/api", "./api-specs");
service.start();

// Files accessible at:
// http://127.0.0.1:8002/docs/index.html
// http://127.0.0.1:8002/api/swagger.json
```

### Restricted File Types

```java
// Only serve web assets
var service = new WebService(
    8002,
    Map.of("/web", "./www"),
    null,
    null,
    false,
    List.of(".html", ".css", ".js", ".png", ".jpg", ".woff2"),  // allowedExtensions
    null,
    100L * 1024 * 1024,
    true);
service.start();
```

### Dynamic Directory Management

```java
var service = new WebService();

// Add directories after construction (remounts immediately if running)
service.addDirectory("/docs", "./documentation");
service.addDirectory("/reports", "./generated/reports");

// Remove a directory
service.removeDirectory("/reports");

service.start();
```

### With Custom Authentication

```java
var service = new WebService(new String[] {"admin", "super-secret-password"});
service.addDirectory("/private", "./sensitive-docs");
service.start();
```

### Binding an Explicit Host and Port

```java
var service = new WebService();
service.addDirectory("/secure", "./secure-files");
// Bind all interfaces on a fixed port; pass 0 for an OS-assigned ephemeral port.
int port = service.start("0.0.0.0", 8002);
```

## Deployment Patterns

### Standalone Service

Run WebService as a dedicated static file server:

```java
import com.signalwire.sdk.web.WebService;

public class WebServer {
    public static void main(String[] args) {
        var service = new WebService();
        service.addDirectory("/docs", "/var/www/docs");
        service.addDirectory("/assets", "/var/www/assets");
        service.addDirectory("/downloads", "/var/www/downloads");
        int port = service.start("0.0.0.0", 8002);
        System.out.println("Serving on port " + port);
    }
}
```

### Alongside AI Agents

`WebService.start()` is non-blocking, so start it first and then run your agent on a different port:

```java
import com.signalwire.sdk.agent.AgentBase;
import com.signalwire.sdk.web.WebService;

public class Main {
    public static void main(String[] args) throws Exception {
        // Start WebService (non-blocking) for documentation.
        var web = new WebService();
        web.addDirectory("/docs", "./agent-docs");
        web.start("0.0.0.0", 8002);

        // Run the agent (blocking) on port 3000.
        var agent = AgentBase.builder()
            .name("My Agent")
            .route("/")
            .port(3000)
            .build();
        agent.run();
    }
}
```

### Docker Deployment

```dockerfile
FROM eclipse-temurin:21-jre

WORKDIR /app

# Copy the built jar (produced by `./gradlew jar`) and static files
COPY ./build/libs/signalwire-sdk-4.0.0.jar /app/signalwire-sdk.jar
COPY ./WebServer.class /app/
COPY ./static /app/static

EXPOSE 8002

CMD ["java", "-cp", "signalwire-sdk.jar:.", "WebServer"]
```

### Systemd Service

Create `/etc/systemd/system/signalwire-web.service`:

```ini
[Unit]
Description=SignalWire Web Service
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/signalwire
Environment="SWML_SSL_CERT_PATH=/etc/ssl/certs/server.crt"
Environment="SWML_SSL_KEY_PATH=/etc/ssl/private/server.key"
ExecStart=/usr/bin/java -cp signalwire-sdk.jar:. WebServer
Restart=always

[Install]
WantedBy=multi-user.target
```

### Nginx Reverse Proxy

For production, use Nginx as a reverse proxy:

```nginx
server {
    listen 80;
    server_name static.example.com;

    # Redirect to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name static.example.com;

    ssl_certificate /etc/ssl/certs/example.com.crt;
    ssl_certificate_key /etc/ssl/private/example.com.key;

    location / {
        proxy_pass http://localhost:8002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Cache static assets
        location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
            proxy_pass http://localhost:8002;
            expires 1h;
            add_header Cache-Control "public, immutable";
        }
    }
}
```

## Best Practices

### Security
1. **Always use HTTPS in production** - Protect data in transit
2. **Change default credentials** - Never use auto-generated auth in production
3. **Restrict file types** - Use `allowedExtensions` to whitelist safe files
4. **Disable directory browsing** - Turn off in production environments
5. **Use reverse proxy** - Put Nginx/Apache in front for additional security

### Performance
1. **Set appropriate cache headers** - WebService adds a 1-hour cache header by default
2. **Limit file sizes** - Adjust `maxFileSize` based on your needs
3. **Use CDN for static assets** - Offload traffic for better performance
4. **Compress large files** - Use gzip/brotli at the reverse proxy level

### Organization
1. **Separate content types** - Use different routes for different file types
2. **Version your assets** - Include version in path (e.g., `/assets/v1/`)
3. **Use index.html** - Provide default files for directories
4. **Document your structure** - Maintain clear directory organization

## Troubleshooting

**Issue: SSL certificate errors** — check the certificate paths:

```java
System.out.println(java.nio.file.Files.exists(java.nio.file.Paths.get("/path/to/cert.pem")));
System.out.println(java.nio.file.Files.exists(java.nio.file.Paths.get("/path/to/key.pem")));
```

**Issue: Permission denied** — ensure read permissions on directories:

```bash
chmod -R 755 /path/to/static/files
```

**Issue: Directory not found** — `addDirectory` throws `IllegalArgumentException` when a path does not exist; use absolute paths:

```java
var service = new WebService();
service.addDirectory("/docs",
    java.nio.file.Paths.get("./documentation").toAbsolutePath().toString());
```

### Debug Logging

Enable debug logging to troubleshoot issues:

```bash
export SIGNALWIRE_LOG_LEVEL=debug
```

## API Reference

### WebService Class

<!-- snippet: no-compile method-signature listing (API reference of WebService constructors, not runnable code) -->
```java
public class WebService {
    public WebService();
    public WebService(String[] basicAuth);
    public WebService(
        int port,
        Map<String, String> directories,
        String[] basicAuth,
        String configFile,
        boolean enableDirectoryBrowsing,
        List<String> allowedExtensions,
        List<String> blockedExtensions,
        long maxFileSize,

```

#### Constructor Parameters
- `port`: Port to bind to (default: 8002)
- `directories`: Map of URL paths to local directories
- `basicAuth`: `[username, password]` for authentication (null -> `SecurityConfig`)
- `configFile`: Path to JSON configuration file (null -> auto-discover)
- `enableDirectoryBrowsing`: Allow directory listing (default: false)
- `allowedExtensions`: List of allowed file extensions
- `blockedExtensions`: List of blocked file extensions
- `maxFileSize`: Maximum file size in bytes (default: 100MB)
- `enableCors`: Enable CORS headers (default: true)

#### Methods

##### start(String host, Integer bindPort) / start()
<!-- snippet: no-compile method-signature listing (API reference of start(), not runnable code) -->
```java
public int start(String host, Integer bindPort);
public int start(); // start("127.0.0.1", port)
```
Start the web service (non-blocking). Binds `bindPort` (0 = OS-assigned) on `host` and returns the actually-bound port.

##### stop()
<!-- snippet: no-compile method-signature listing (API reference of stop(), not runnable code) -->
```java
public void stop();
```
Stop the service and release the socket. Safe to call when not running.

##### addDirectory(String route, String directory)
<!-- snippet: no-compile method-signature listing (API reference of addDirectory(), not runnable code) -->
```java
public void addDirectory(String route, String directory);
```
Add a new directory to serve. Remounts immediately if the server is already running; throws `IllegalArgumentException` when the path does not exist or is not a directory.

<!-- snippet: no-compile method-signature listing (API reference of removeDirectory(), not runnable code) -->
```java
public void removeDirectory(String route);
```

Remove a directory from being served (no-op when absent).

## Integration with SignalWire Agents

WebService complements AI agents by providing static file serving:

```java
import com.signalwire.sdk.agent.AgentBase;
import com.signalwire.sdk.swaig.FunctionResult;
import com.signalwire.sdk.web.WebService;
import java.util.Map;

public class DocumentationAgentApp {
    public static void main(String[] args) throws Exception {
        var agent = AgentBase.builder()
            .name("Documentation Assistant")
            .route("/")
            .port(3000)
            .build();

        // Reference documentation served by WebService.
        agent.promptAddSection(
            "Documentation",
            "User documentation is available at https://example.com:8002/docs/");

        agent.defineTool("get_doc_link", "Get link to a documentation page",
            Map.of("type", "object", "properties", Map.of(
                "doc_name", Map.of("type", "string", "description", "Name of the documentation page"))),
            (toolArgs, raw) -> {
                String docName = (String) toolArgs.get("doc_name");
                return new FunctionResult(
                    "Documentation available at: https://example.com:8002/docs/" + docName + ".html");
            });

        // Start WebService (non-blocking) for documentation.
        var web = new WebService();
        web.addDirectory("/docs", "./documentation");
        web.start("0.0.0.0", 8002);

        // Run the agent (blocking) on port 3000.
        agent.run();
    }
}
```

## Summary

WebService provides a secure, configurable static file server that integrates with the SignalWire AI Agents SDK for Java. Built on the JDK `HttpServer`, its non-blocking `start()` makes it easy to run standalone or alongside your AI agents while providing configurable security features and flexible deployment options.
